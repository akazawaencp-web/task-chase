# タスクチェイスシステム 設計書

## コンセプト
「LINEに送るだけで、調べて、追いかけて、完了まで面倒を見てくれるAIアシスタント」

## 基本フロー

1. RIOがLINEにタスクを送る（音声入力アプリ経由）
2. AIがタイトル・補足・期限を自動分解（タイトルは一言に要約、長文は説明欄へ）
3. Googleカレンダーにタスク登録
4. AIが詳細調査（Sonnet使用、Web検索+YouTube含む）
   - 5W1Hで要素分解
   - 依存関係を2段階以上チェック
   - 所要時間・費用を自動見積もり
   - やらなかった場合のリスクを明示
   - 行動が必要なタスクは日程提案付き
   - 面倒ポイントを予測して解消策を先回り提示
   - 着手ハードルを下げる分解提案
5. 調査結果HTMLを生成 → GitHub Pagesにアップ
6. LINEに概要+HTMLリンク+日程提案を送信
7. 毎日チェイス（Haiku使用）
8. RIOがLINEで「やった」or 番号指定で完了報告
9. Googleカレンダーのタスク完了 + 「次はこれ」提案

## チェイスルール

| 状況 | 頻度 |
|---|---|
| 期限なし | 毎日1回 |
| 期限あり（3日以上先） | 毎日1回 |
| 期限あり（3日以内） | 朝・夕の2回 |
| 期限あり（当日） | 朝・昼・夕の3回 |
| 「あとでやる」を3回連続 | 「本当にやる？やめる？」と確認 |
| 2週間以上放置 | 「これまだいる？」と棚卸し提案 |
| 3日放置 | 「何がブロッカー？」と質問 |

## LINEリッチメニュー（6ボタン）

| ボタン | 動作 |
|---|---|
| タスク追加 | テキスト入力画面 |
| 今日のタスク | 完了しやすい順で表示 |
| 完了報告 | タスク番号指定で完了 |
| 今週のまとめ | 週次レビュー表示 |
| あとでやる | 翌日リスケ |
| タスク一覧 | 全タスク期限順表示 |

## 定期レポート

| タイミング | 内容 |
|---|---|
| 毎日朝 | 今日のタスク（完了しやすい順）+「今日はこれだけはやろう」の1件 |
| 毎週日曜 | 週次レビュー（完了数、放置数、来週の予定） |
| 毎月末 | 月次サマリー + API費用の内訳 |

## カレンダー連携

| 機能 | 動作 |
|---|---|
| 予定からタスク逆算 | 1週間先までの予定をチェック、準備が必要なら事前タスク提案 |
| 日程提案 | カレンダーの空きを見て「ここでやれば？」と提案 |
| 習慣タスク | 繰り返しタスクを自動登録 |
| 年間カレンダー | 年次タスクを登録、時期が来たら自動でタスク化。随時追加可 |

## タスク完了時の動作

- Googleカレンダーのタスクを完了
- 「次はこれ」提案（勢いがある時に畳みかける）
- 完了実績を記録（週次・月次レポート用）

## 調査結果HTMLの構成

1. 次の一歩（大きく表示）
2. やることチェックリスト（依存関係含む、2段階以上）
3. 所要時間・費用の見積もり
4. やらなかった場合のリスク
5. 面倒ポイントと解消策
6. 詳細情報（折りたたみ式）
7. エビデンス（あれば）

## HTMLデザイン方針

- テーマカラー: コーラル #ea8768 × スカイブルー #33b6de
- 左側角丸なし（border-radius: 2px 10px 10px 2px）
- シンプル優先、装飾より実用性
- 絵文字アイコン絶対NG（CSSドットで代替）
- スマホで読みやすいレスポンシブデザイン
- 記事を読むような体験を目指す

## モデル使い分け

| 処理 | モデル | 理由 |
|---|---|---|
| タスクの詳細調査 | Sonnet | 品質重視 |
| チェイス通知の生成 | Haiku | 定型文中心、コスト抑制 |
| LINE内のやり取り | Sonnet | 質問への的確な回答が必要 |
| タスク完了処理 | Haiku | 単純処理 |

## 費用見込み

- 月800〜1,400円程度（3,000円以下が条件）
- 月末にLINEでAPI費用の内訳を報告

## 技術構成

| 部品 | 役割 |
|---|---|
| LINE Messaging API | LINEの送受信+リッチメニュー |
| Google Calendar API | タスクの読み書き+予定の参照 |
| Claude API（Sonnet/Haiku） | 調査・文章生成・やり取り |
| Web検索 | タスクの詳細調査 |
| GitHub Pages | 調査結果HTMLのホスティング |
| スケジューラー | 毎日のチェイス・週次月次レポート |
| 費用トラッカー | API呼び出しごとのコスト記録 |

## 入力仕様

- 音声入力アプリ（Aqua Voice / Type-less）からLINEに送信
- AIがタイトルと説明を自動切り分け（タイトルは一言に要約）
- 期限は自然言語で指定（「来週まで」「4月22日まで」等）
- 期限なしの場合は「期限なし」として登録
- 補足なしでも動作する（AIが一般的な項目を自動調査）
- 補足ありの方が調査精度が上がる

## チェイスの工夫

- トーン日替わり（慣れてスルーしない対策）
- 完了しやすい順（5分以内のもの優先）で通知
- 3日放置で「何がブロッカー？」と質問
- 着手ハードルを下げる分解提案
- 面倒ポイントの先回り解消
